Board-sync CLI (`npm run set-project-status`) - the exact surface the agent-loop
workflows invoke. Demonstrates the hard requirement: best-effort and non-fatal.
The only non-zero exit is a usage error (bad args); every operational
skip/failure logs a clear message and exits 0 so it can never fail a CI job.

### 1) PROJECTS_TOKEN unset -> logged no-op, exit 0
$ GITHUB_REPOSITORY=puopg/tic-tac-toe npm run --silent set-project-status -- --issue 26 --status "In Progress"
set-project-status: skipped - PROJECTS_TOKEN is not set. Board sync is optional.
[exit code: 0]

### 2) Token set but owner/name unresolvable -> logged no-op, exit 0
$ PROJECTS_TOKEN=dummy GITHUB_REPOSITORY="" npm run --silent set-project-status -- --issue 26 --status "In Review"
set-project-status: skipped - could not resolve owner/name from --repo or GITHUB_REPOSITORY.
[exit code: 0]

### 3) --help -> usage, exit 0
$ npm run --silent set-project-status -- --help
usage: set-project-status --issue N --status "In Progress" [--repo owner/name]
[exit code: 0]

### 4) Real GitHub GraphQL call with a bogus token -> HTTP 401 degrades to logged no-op, exit 0
$ PROJECTS_TOKEN=ghp_invalidtoken000 npm run --silent set-project-status -- --issue 26 --status "In Progress" --repo puopg/tic-tac-toe
set-project-status: skipped - GraphQL error: HTTP 401 Unauthorized.
[exit code: 0]

### 5) Usage error: missing --issue -> stderr + exit 2 (the only non-zero path)
$ PROJECTS_TOKEN=dummy npm run --silent set-project-status -- --status "In Progress"
set-project-status: --issue must be a positive integer
[exit code: 2]
